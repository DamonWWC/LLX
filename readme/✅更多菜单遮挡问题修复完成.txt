订单管理更多菜单遮挡问题修复完成

🔍 问题分析
- 最后一个订单点击更多按钮时，菜单向下展开被滚动容器边界遮挡
- 原因：更多菜单使用固定向下展开（top: 100%），没有考虑容器边界
- 影响：用户无法看到和操作最后一个订单的更多菜单选项

🔧 解决方案

1. 智能定位检测
   - 添加 checkMenuPosition() 函数检测订单位置
   - 计算订单底部到列表底部的距离
   - 当距离小于菜单高度时，自动向上展开

2. CSS样式优化
   - 增加 .more-menu.upward 类用于向上展开
   - 提高 z-index 从 100 到 1000，确保层级正确
   - 添加向上展开的动画效果

3. 动态类名绑定
   - WXML模板中添加 {{order.menuUpward ? 'upward' : ''}} 条件类
   - 根据检测结果动态切换展开方向

📱 技术实现

1. JavaScript逻辑
   ```javascript
   checkMenuPosition(orderId) {
     // 使用 wx.createSelectorQuery() 获取元素位置
     // 计算距离并决定展开方向
     // 更新订单的 menuUpward 属性
   }
   ```

2. CSS样式
   ```css
   .more-menu.upward {
     top: auto;
     bottom: 100%;
     transform: translateY(10rpx);
   }
   ```

3. 模板绑定
   ```xml
   <view class="more-menu {{order.showMoreMenu ? 'show' : ''}} {{order.menuUpward ? 'upward' : ''}}">
   ```

✅ 修复效果

1. 智能展开
   - 顶部订单：菜单向下展开（原有行为）
   - 底部订单：菜单向上展开（新增功能）
   - 中间订单：根据空间自动选择方向

2. 用户体验提升
   - 所有订单的更多菜单都能完整显示
   - 流畅的展开动画效果
   - 保持一致的交互体验

3. 兼容性保证
   - 不影响现有功能
   - 保持所有订单操作正常
   - 响应式适配良好
